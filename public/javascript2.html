JavaScript - Variables

JavaScript variables can be declared in several ways:

  • Automatically (Implicit Declaration)
  • Using var
  • Using let
  • Using const

It is considered good programming practice to always declare variables before use.

The var keyword was used in all JavaScript code from 1995 to 2015.

The let and const keywords were introduced in ECMAScript 2015 (ES6).

The var keyword should only be used in code intended for older environments where let and const are not supported.

When to use var, let, or const:

  • 1. Always declare variables.
  • 2. Use const if the value should not be changed.
  • 3. Use const if the reference type (like arrays and objects) should not be changed.
  • 4. Use let if you need a variable whose value might change.
  • 5. Use var only if you must support very old browsers.

All JavaScript variables must be identified with unique names called identifiers.

Identifiers can be short names (like x and y) or more descriptive names (like age, sum, totalVolume).

The general rules for constructing variable names (identifiers) are:

  • Names can contain letters, digits, underscores, and dollar signs.
  • Names must begin with a letter, underscore (_), or dollar sign ($).
  • Names are case-sensitive (e.g., y and Y are different variables).
  • Reserved words (like JavaScript keywords) cannot be used as names.

The equality operator in JavaScript is ==, but it is recommended to use === for strict equality comparison.

JavaScript variables can hold numbers like 100 and text values (strings) like "John Doe".

In programming, text values are called strings.

JavaScript handles various data types, but for now, consider numbers and strings. Strings are written inside quotes (single or double), and numbers are written without quotes.

If a number is placed in quotes, it is treated as a string.

You cannot re-declare a variable declared with let or const in the same scope.

Using the dollar sign ($) is not very common in general JavaScript code but is used in libraries like jQuery, where $ is used to select HTML elements (e.g., $("p") selects all <p> elements).

The underscore (_) is not commonly used in JavaScript, but some developers use it as a convention for private or hidden variables.

JavaScript - Let

The

let
keyword was introduced in ES6 (2015).

Variables defined with

let
:

  • Cannot be redeclared.
  • Must be declared before use.
  • Have block scope.

Before ES6 (2015), JavaScript had only global scope and function scope.

ES6 introduced two important new JavaScript keywords:

let
and
const
, which provide block scope in JavaScript.

Variables declared inside a

{ }
block cannot be accessed from outside the block.

Variables declared with the

var
keyword do NOT have block scope.

Variables defined with

var
are hoisted to the top of their function or global scope and can be initialized at any time.

Variables defined with

let
are also hoisted to the top of their block, but are not initialized.

This means that using a

let
variable before it is declared will result in a
ReferenceError
.

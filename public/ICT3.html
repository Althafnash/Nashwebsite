System Development Life Cycle

A system is a collection of components that interact together to achieve a specific task.

A system converts data into information.

Manual System

  • Errors are minimal when data is processed by a computer program.
  • Since data is inserted manually, there is more room for error.
  • Processing of data is less efficient.
  • Requires a large space for storage of data.
  • Data is open to many threats.

Advantages of Using a Computer-Based System

  • Errors are minimal when data is processed by a computer program.
  • Data can be processed more efficiently.
  • Large amounts of data can be stored on a small physical surface such as a database.
  • Security can be ensured with backups and the use of passwords.

Stages of the System Development Life Cycle

  • Identification of requirements
  • Designing the solution
  • Coding of the solution
  • Testing and debugging
  • Deployment of the system
  • Maintenance of the system

Ways of Identifying Requirements

  • Observing
  • Interviews
  • Questionnaires
  • Document sample collection
  • Prototyping

Key Points in Designing Software

  • Identification of software and software architecture
  • User interface
  • Identification of main hardware system and its components
  • Identification of dependencies of each subsystem
  • Deciding on the hardware and software to run on the computer
  • Designing infrastructure for the software, database, and user interface
  • Planning of tests

The next step is to code the system using a suitable programming language.

The main objective of testing and debugging is to minimize errors from the previous steps.

Phases of Testing

  • Unit testing
  • Integration testing
  • System testing
  • Acceptance testing

Unit testing involves testing each unit of the system to ensure that it functions correctly with the given inputs and produces the expected outputs.

Integration testing involves combining tested and debugged units to form the complete system and testing the integrated system.

System testing involves testing the entire system to ensure it meets the specified requirements and functions correctly with provided inputs.

Acceptance testing involves validating the system with end users to ensure it meets their needs and requirements.

Types of Deployment

  • Direct deployment: The existing system is completely terminated and replaced with the new system.
  • Parallel deployment: The existing system and the new system are used concurrently.
  • Pilot deployment: The new system is implemented in a selected area before full-scale deployment.
  • Phased deployment: The new system is deployed in stages, with each stage leading to the next until the entire system is implemented.

Once a system is installed and in use for a certain period, it requires maintenance. System maintenance may involve:

  • Adopting new technology to improve efficiency.
  • Addressing issues identified during the SDLC.
  • Developing the system to meet new requirements.

There are several models for the SDLC, each facilitating system development in different ways. Examples of these models include:

  • Waterfall model
  • Iterative incremental model
  • Prototyping model
  • Spiral model

The Waterfall model consists of several stages of linear development:

  • Identify requirements
  • Complete one stage before proceeding to the next
  • The result of the developed system is found at the final stage

The Iterative Incremental model allows software developers to build on what was learned during development, using it to improve the system. In this model, the system is developed through repeated cycles and in smaller portions at a time.
